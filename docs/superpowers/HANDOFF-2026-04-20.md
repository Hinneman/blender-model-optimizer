## Open problem

Holes appear in the decimated output on AI-generated meshes (specifically a test mesh with a camo-textured cover draped over oil barrels). The holes appear in the cover even when the entire Decimate step is minimized:

- `Protect UV Seams`: OFF
- `Planar Post-Pass`: OFF
- `Passes`: 1
- `Decimate ratio`: 0.1

...so neither the 1.8.0 `Planar Post-Pass` nor the seam-protection toggle is the cause. Something further upstream in the pipeline is removing cover faces.

The user reports these holes did not appear in earlier versions (pre-1.8.0). They are now visible because this session's work reduced decimation-era texture/geometry artifacts, exposing a pre-existing regression.

## Prime suspect for the holes

**`Remove Interior` → Ray Cast method** in [src/geometry.py](../../src/geometry.py) (`_remove_interior_raycast`, line ~164).

Reasoning:
- Ray Cast is default ON.
- It casts rays outward from each face and marks a face as interior when all rays hit back-faces.
- A draped cover has many concave folds. From certain faces inside a fold, all outward rays do hit back-faces (of the opposite fold surface, or the nearby barrel below).
- Result: cover faces falsely classified as interior → deleted → holes.
- The 1.7.1 change *widened* the ray-cast cone (from ~6° to ~55° with 13 rays) specifically to reduce false positives on concave regions. That change helped on the fuselage-spine AI aircraft case the 1.7.1 CHANGELOG mentions, but may not have gone far enough for draped thin-shell geometry like the camo cover.

## Suggested next step

**Diagnostic test** (before committing to any fix): ask the user to run the full pipeline on the camo mesh with `Remove Interior` OFF and report whether the cover comes out whole.

- If cover is clean with Remove Interior off → Remove Interior is the culprit; scope work to tightening that step's false-positive rate. Options:
  - Require N of M rays (not all rays) to hit back-faces before deleting — a soft quorum instead of unanimous.
  - Skip faces whose area is above a threshold (thin-shell cover faces tend to be larger than the fuselage-interior noise the step was designed to catch).
  - Skip faces whose vertices all lie on the convex hull — these are almost certainly exterior regardless of ray hits.
  - Detect thin-shell geometry and use the `Enclosed Parts` (loose-parts) method instead of Ray Cast for such meshes.
- If cover still has holes with Remove Interior off → investigate earlier in the pipeline (`Fix Geometry`'s `print3d_clean_non_manifold` / manual fill-holes fallback, or the `dissolve_degenerate` pre-pass). Unlikely but possible.

## What was tried this session and abandoned

Two attempts to eliminate post-decimate texture bleed on clean-UV meshes (barrel rims/lids). Both produced worse side effects than the problem they solved.

### Attempt 1: Hard topological seam split + restitch (abandoned)

Spec: [specs/2026-04-19-hard-uv-seam-constraint-design.md](specs/2026-04-19-hard-uv-seam-constraint-design.md)
Plan: [plans/2026-04-19-hard-uv-seam-constraint.md](plans/2026-04-19-hard-uv-seam-constraint.md)

Idea: split the mesh along UV seams before decimation so COLLAPSE cannot cross them, re-weld after. On fragmented-UV meshes the restitch couldn't re-weld boundaries that drifted too far apart during collapse, shredding the mesh into disconnected flaps. Both docs have an "ABANDONED (2026-04-20)" banner at the top.

### Attempt 2: Vertex-group seam weighting (this session, likely revert pending)

Spec: [specs/2026-04-20-vertex-group-seam-weighting-design.md](specs/2026-04-20-vertex-group-seam-weighting-design.md)
Plan: [plans/2026-04-20-vertex-group-seam-weighting.md](plans/2026-04-20-vertex-group-seam-weighting.md)

Idea: build `AIOPT_Seam_Protect` vertex group with seam-endpoint + one-ring weighted at 1.0 and everything else at 0.1; each COLLAPSE modifier references it with `invert_vertex_group=True`. Mesh topology stays connected (no flap-shredding), and on clean-UV meshes the barrel rim/lid bleed is fully eliminated. User-reported regression: small sawtooth-edge holes scattered on the camo cover even with the toggle ON — attributed to weight-bias causing isolated protected strips that survive collapse but fail to connect, then get removed by `delete_loose` in the post-decimate cleanup.

**Key correction discovered at the very end:** the same holes appear with the toggle OFF and Planar Post-Pass OFF, which means they predate the vertex-group change. The seam-weighting change is not the root cause. See "Prime suspect for the holes" above.
